# دليل تثبيت رقيم — رقيم الحضور الجامعي الذكي 🎓
منصة إدارة الحضور الجامعي الرقمية القائمة على الاشتراك ومسح رموز الـ QR للطلاب دون حسابات.

---

## 🚀 نظرة عامة عن المشروع

مشروع **رقيم** يهدف إلى أتمتة وتسجيل حضور وغياب الطلاب في الجامعات العربية.
- **المدير العام (Super Admin):** يدير هيكلية الكليات، ويسجل الأساتذة، ويستورد كشوف الطلاب عبر Excel/CSV مولداً بطاقات الحضور.
- **الأستاذ (Professor):** يدير المحاضرات، ويفتح جلسات الحضور باستخدام الكاميرا لمسح كروت الطلاب لحظياً.
- **الطالب:** لا يملك حساباً، بل يستلم بطاقة حضور تحتوي على رمز QR فريد عبر بوت تيليجرام، ويقوم بإظهارها للأستاذ (أوفلاين بالكامل دون إنترنت).

---

## 🛠️ التقنيات المستخدمة

- **الواجهة:** React.js + Vite + CSS Modules (RTL كامل باللغة العربية مع خط Tajawal)
- **قاعدة البيانات وسحابة الخفاء:** Supabase (PostgreSQL + Auth + Storage + Realtime)
- **توليد الـ QR:** مكتبة `qrcode`
- **مسح الـ QR:** مكتبة `html5-qrcode`
- **بوت التيليجرام:** خدمة Node.js منفصلة تعمل بمكتبة `node-telegram-bot-api`
- **التقارير والتصدير:** مكتبتا `xlsx` و `jspdf` / `jspdf-autotable`

## 📦 خطوات التثبيت والتشغيل

### 1. إعداد قاعدة بيانات Supabase

1. أنشئ مشروعاً جديداً على [Supabase](https://supabase.com).
2. افتح نافذة الاستعلامات **SQL Editor** في لوحة تحكم Supabase.
3. انسخ محتويات ملف [schema.sql](file:///d:/1.%20Osama/6.%20Companies/Raqim/schema.sql) بالكامل والصقها في المحرر ثم انقر على **Run**.
   - *تقوم هذه العملية بتهيئة كافة الجداول والقيود، ودوال الأمان (RPC)، وتفعيل سياسات الحماية الذاتية (RLS)، بالإضافة إلى بث البيانات الفوري (Realtime) لجدولي الحضور وطلبات البوت.*
4. انتقل إلى تبويب **Storage** وأنشئ سلة تخزين (Bucket) جديدة باسم:
   - **الاسم:** `qr-cards`
   - **الخصوصية:** اجعلها **Public** (عامة ومكشوفة للجميع) ليتمكن البوت والطلاب من قراءة البطاقات.

---

### 2. إعداد وتشغيل تطبيق الويب (React Frontend)

1. انتقل إلى المجلد الرئيسي للمشروع.
2. قم بإنشاء ملف `.env` بناءً على ملف [.env.example](file:///d:/1.%20Osama/6.%20Companies/Raqim/.env.example):
   ```bash
   VITE_SUPABASE_URL=رابط_مشروع_سوبابيس_الخاص_بك
   VITE_SUPABASE_ANON_KEY=المفتاح_العام_الخاص_بالمشروع
   VITE_SUPER_ADMIN_EMAIL=admin@raqim.com
   ```
3. قم بتثبيت الاعتماديات:
   ```bash
   npm install
   ```
4. شغل خادم التطوير المحلي:
   ```bash
   npm run dev
   ```
5. לבناء نسخة الإنتاج، استخدم:
   ```bash
   npm run build
   ```

---

### 3. إعداد وتشغيل بوت التيليجرام (Node.js Bot Service)

1. اذهب إلى مجلد البوت الفرعي:
   ```bash
   cd bot
   ```
2. أنشئ ملف بيئة عمل للبوت باسم `.env` أو ضع المتغيرات في البيئة العامة للجهاز:
   ```bash
   VITE_SUPABASE_URL=رابط_مشروع_سوبابيس_الخاص_بك
   SUPABASE_SERVICE_ROLE_KEY=المفتاح_الخدمي_الخاص_بسوبابيس_لتخطي_الـ_RLS
   TELEGRAM_BOT_TOKEN=توكن_البوت_الذي_أعطاك_إياه_BotFather
   ```
   > ⚠️ **تحذير أمان:** لا تقم أبداً بإرفاق مفتاح `SUPABASE_SERVICE_ROLE_KEY` في فرونت إند التطبيق أو رفعه لمستودع عام، احتفظ به للبيئة السحابية للفرونت والبوت بشكل مخفي وسري!
3. قم بتثبيت المكونات الخاصة بالبوت:
   ```bash
   npm install
   ```
4. ابدأ تشغيل خدمة البوت:
   ```bash
   npm start
   ```

## 🎯 دليل دورة العمل التجريبية

1. **الولوج كأدمن:** سجل الدخول باستخدام بريد المدير الافتراضي المذكور بملف `.env` (مثل `admin@raqim.com`) وكلمة المرور التي تنشئها له. ستفتح لك لوحة التحكم الكاملة.
2. **التهيئة الأكاديمية:** اذهب لصفحة "الجامعات" وأنشئ جامعة، ثم كلية، ثم مادة علمية وخصصها لمرحلة معينة.
3. **تسجيل أستاذ:** اذهب لصفحة "الأساتذة" وأنشئ حساباً لأستاذ وعين له جامعته وتاريخ انتهاء اشتراكه بالمستقبل، ثم اسند إليه المادة التي أنشأتها للتو.
4. **استيراد الطلاب:** في صفحة "الطلاب"، اختر الجامعة، وقم برفع ملف طلاب CSV يحتوي على كشف أسمائهم وأرقامهم الأكاديمية. سينشئ النظام بطاقات الحضور تلقائياً ويرفعها لسلة التخزين.
5. **تفعيل البوت:** يدخل الطالب لبوت التيليجرام الخاص بك، يكتب أمره البسيط `/start` ثم يرسل رقمه الجامعي، فيقوم البوت بالتحقق من قاعدة البيانات وإرسال الكارت له فوراً، كما يقوم بتخزين رقم الـ `chat_id` الخاص به لتفعيل ميزة إعادة الإرسال اللحظية من لوحة المشرف.
6. **بدء تسجيل الحضور:** يسجل الأستاذ دخوله للتطبيق، ويختار مادته ويبدأ الجلسة. ستفتح كاميرا الأستاذ والعداد الفوري (Realtime).
7. **المسح الفوري والخصوصية:** يوجه الطالب كود الـ QR الخاص به أمام كاميرا الأستاذ. يُظهر النظام فوراً اسماً متحركاً للطالب باللون الأخضر مع عداد فوري، وإذا تكرر المسح سيرفضه النظام، وتظهر البيانات لحظياً في القائمة السفلية للأستاذ.
