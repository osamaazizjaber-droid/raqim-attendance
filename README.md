# دليل تثبيت رقيم — رقيم الحضور والنتائج الجامعية 🎓

منصة **رَقِيم** هي منصة إدارة حضور واختبارات جامعية متعددة الصلاحيات (SaaS) متكاملة، تعمل باللغة العربية وتدعم التوجيه التلقائي للمستخدمين وإدارة الاشتراكات السنوية الفردية والمؤسساتية على مستوى الكلية. يتيح النظام تسجيل الحضور بواسطة مسح رموز الـ QR للطلاب دون الحاجة لحسابات طلابية، وتوزيع كروت الحضور والنتائج عبر بوت تيليجرام تفاعلي محمي بـ CAPTCHA حسابية، مع توليد شهادات النتائج الرسمية وتنزيلها مضغوطة.

---

## 🏛️ صلاحيات النظام والتحقق (3 أدوار)

1.  **المشرف العام (Super Admin):**
    *   إنشاء وتعديل وحذف الكليات الجامعية، وتعيين تاريخ انتهاء اشتراكاتها السنوية مباشرة.
    *   تفعيل حسابات مدراء الكليات (College Admins) بشكل فوري.
    *   الاطلاع على الإحصائيات الشاملة لكافة الكليات في المنصة.
2.  **مدير الكلية (College Admin):**
    *   تهيئة الأقسام، المراحل، والمواد الدراسية (وتخصيص عدد الوحدات لكل مادة).
    *   تسجيل الأساتذة وإسناد المواد لهم وتجديد اشتراكاتهم الشخصية.
    *   استيراد الطلاب وتوليد بطاقات الـ QR (بجودة Retina عالية التباين) وتعيين مواد الإعادة.
    *   ترحيل الطلاب جماعياً بين المراحل مع حفظ كروت الـ QR وسجلات الحضور والتاريخ.
    *   رفع الدرجات، وتوليد كشوف شهادات PDF (بمقياس الوزارة العراقي)، وتنزيلها دفعة واحدة كملف ZIP.
    *   الاطلاع على إحصائيات كليته، ومحجوب تلقائياً في حال انتهاء اشتراك الكلية.
3.  **الأستاذ (Professor):**
    *   بدء جلسة الحضور، وفتح كاميرا المسح مع فلترة مخصصة للطلاب المعيدين (Badge إعادة).
    *   الاطلاع على تقارير حضور وغياب الجلسات الخاصة بمواده.
    *   تلقي تقرير حضور تفصيلي لحظي عبر بوت تيليجرام فور إغلاق الجلسة بالمنصة.

## 🛠️ التقنيات المستخدمة

*   **الواجهة:** React.js + Vite + CSS Modules (تصميم RTL داكن زجاجي متناسق مع خط Tajawal)
*   **قاعدة البيانات والسحاب السري:** Supabase (PostgreSQL + Auth + Storage + Realtime)
*   **توليد الـ QR:** مكتبة `qrcode`
*   **مسح الـ QR:** مكتبة `html5-qrcode`
*   **ضغط الملفات:** مكتبة `jszip`
*   **توليد التقارير:** مكتبتا `xlsx` و `jspdf` / `jspdf-autotable`
*   **البوت الخلفي:** خدمة Node.js منفصلة تعمل بمكتبة `node-telegram-bot-api`

## 📦 خطوات التثبيت والتشغيل بالتفصيل

### 1. إعداد قاعدة بيانات Supabase

1.  أنشئ مشروعاً جديداً على منصة [Supabase](https://supabase.com).
2.  افتح تبويب **SQL Editor** في لوحة تحكم مشروعك.
3.  انسخ محتويات ملف [schema.sql](file:///d:/1.%20Osama/6.%20Companies/Raqim/schema.sql) بالكامل، والصقها في المحرر ثم اضغط **Run**.
    *   *تقوم هذه العملية بإنشاء كافة الجداول، والقيود الفريدة لحماية البيانات من التكرار، وتنشيط الـ Realtime للبث اللحظي للدردشات وحضور الطلاب وجلسات الأساتذة، وتغذية جداول التقديرات والمراحل.*
4.  انتقل إلى تبويب **Storage** وأنشئ سلتين للتخزين (Buckets) كالتالي:
    *   **الأولى:** باسم `qr-cards` واجعلها **Public** (عامة).
    *   **الثانية:** باسم `certificates` واجعلها **Public** (عامة) لرفع شهادات درجات الطلاب.

### 2. إعداد وتشغيل تطبيق الويب (React Frontend)

1.  افتح المجلد الرئيسي للمشروع في مبنى الأوامر.
2.  قم بتهيئة ملف البيئة `.env` بالاستناد إلى ملف [.env.example](file:///d:/1.%20Osama/6.%20Companies/Raqim/.env.example):
    ```bash
    VITE_SUPABASE_URL=https://your-project-id.supabase.co
    VITE_SUPABASE_ANON_KEY=your-anon-public-key
    VITE_SUPER_ADMIN_EMAIL=osamaazizjaber@gmail.com
    ```
3.  قم بتثبيت الاعتماديات والمكتبات المطلوبة:
    ```bash
    npm install
    ```
4.  شغّل خادم التطوير المحلي:
    ```bash
    npm run dev
    ```
5.  لبناء نسخة إنتاج نظيفة متوافقة، استخدم:
    ```bash
    npm run build
    ```

### 3. إعداد وتشغيل خدمة البوت (Node.js Telegram Bot)

1.  اذهب إلى مجلد البوت:
    ```bash
    cd bot
    ```
2.  تأكد من مطابقة ملف `.env` الخاص بالبوت بالبيانات السرية (يتخطى حظر الـ RLS):
    ```bash
    VITE_SUPABASE_URL=https://your-project-id.supabase.co
    SUPABASE_SERVICE_ROLE_KEY=your-service-role-key-bypasses-rls
    TELEGRAM_BOT_TOKEN=token-obtained-from-botfather
    ```
    > ⚠️ **تحذير أمان:** لا تضع مفتاح `SUPABASE_SERVICE_ROLE_KEY` في الجزء الأمامي من كود التطبيق نهائياً.
3.  قم بتثبيت المكونات الخاصة بالبوت:
    ```bash
    npm install
    ```
4.  شغل خدمة البوت:
    ```bash
    npm start
    ```

## 🎯 دليل دورة العمل والاستعلام

1.  **المشرف العام:** يسجل دخوله ببريد الأدمن الافتراضي، وينشئ الكلية المتعاقدة مباشرة (مثلاً: "كلية دجلة الجامعة") ويحدد تاريخ صلاحية اشتراكها السنوي. ثم ينشئ حساب مدير لتلك الكلية ويفعله فوراً.
2.  **مدير الكلية:** يسجل دخوله للكلية:
    *   يهيئ الأقسام والمواد الدراسية والوحدات.
    *   يسجل الأساتذة ويسند لهم المواد.
    *   يستورد كشوف الطلاب عبر رفع ملف CSV.
    *   ينشئ ويضبط مواد الإعادة لبعض الطلاب المتأخرين.
3.  **توزيع كروت الـ QR للطلاب:**
    *   يدخل الطالب للبوت ويرسل **رقمه الجامعي** (e.g. `2023/CS/0142`).
    *   يتحقق البوت، ويرسل بطاقة الحضور الرسمية كصورة، ويحفظ `telegram_chat_id` لتأكيد تفعيل الحساب.
4.  **تسجيل الحضور الفعلي:**
    *   يسجل الأستاذ دخوله، ويفتح جلسة لمادته، وتفتح كاميرا المسح.
    *   يوجه الطلاب كروت الـ QR لهواتفهم (دون الحاجة لإنترنت بالكامل).
    *   تتحقق الكاميرا لحظياً؛ إذا تكرر المسح تمنعه، وتحدث الأعداد لحظياً بالـ Realtime مع إظهار شارات "إعادة" للطلاب المعيدين.
    *   عند إنهاء الجلسة، يتلقى الأستاذ كشفاً تفصيلياً (حاضرين وغائبين) تلقائياً عبر البوت.
5.  **بوابة الاستعلام العام عن النتائج:**
    *   **عبر البوت:** يكتب الطالب أمر `/نتائج` أو "نتائجي"، يطلب منه البوت اسمه الثلاثي الكامل، ويولد البوت سؤال أمان حسابي (CAPTCHA) جمع بسيط لمنع الروبوتات، ثم يعرض له كشف الدرجات برسالة نصية منسقة.
    *   **عبر الويب:** يدخل الطالب لصفحة `/results` العامة بدون تسجيل دخول، ويدخل رقمه الجامعي وحل الكابتشا الحسابية المحدثة، فتظهر له قائمة الدرجات والتقديرات بألوانها الرسمية.
